Porting

1. INTRODUCTION

CLI parser is mostly platform-independent. However, there are couple areas that needs to be rewritten/modified when porting to different platforms - some basic data types and I/O subsystem

2. BASIC DATA TYPES

CLI Parser assumes the following types are defined - int32_t, uint32_t, int16_t, uint16_t, uint8_t, int8_t. In Linux / MAC OS X, stdint.h defines these types. If your platform does not have stdint.h, you will need to define these types in cparser.h.

2. I/O SUBSYSTEM

I/O subsystem refers to a set of functions that receives keystrokes from users and sends output to user's screen. The default I/O assumes input comes from a UNIX terminate as stdin and output goes to stdout. If these are not true, you need to provide a set of functions to initialize, cleanup I/O subsystem, to read a character and to write a character and a string. cparser_io.h provides the function prototype of all I/O functions that needs to be rewritten when porting to a different platforms.
